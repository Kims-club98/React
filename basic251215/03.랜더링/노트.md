## 리엑트와 리렌더링
: 리엑트의 장점이 리렌더링에 있다.

### 실습1. render02.html
  => 바닐라JS(JavaScript) --> 변경으로 인해 Element를 다시 그리고 있었다...
  비교대상 render03.html <br>
  리액트 -> 변경된 부분만 다시 그림 (효과적!!) <br>

  이런게 가능한 이유??? <br>
  바뀐 걸 알려면 계속 비교를 해야 한다. -> trade off다...
  이것은 성능의 문제가 없다면 효과적일 것이다... <br>

  React앨리먼트는 불변객체(immutable)이다...(==변하지 않는 객체)
  ReactDOM.render(element, foot) 전달할 뿐, 변경 판단 및 반영은 리액트가 알아서 한다.
  즉))) 변경사항 반영은 리엑트에게 일임한다!!

  앨리먼트를 생성한 이후 -> 해당 앨리먼트의 자식이나 속성은 변경 不可함!

  리액트는 "비교 알고리즘을 가지고 있다."
  IF) 앨리먼트의 타입이 바뀐다면? => 이전 엘리먼트는 버리고 새로 그린다!
  IF) 반대로 앨리먼트의 타입이 같으면? => key를 먼저 비교 후 props를 비교하여 변경사항을 반영함!!

  재조정(Reconciliation)
    출: https://ko.legacy.reactjs.org/docs/reconciliation.html

    DOM이 계속 들어오면 이전 버츄얼 돔과 한 번 더 비교하고 그것을 반영함.
    같은 type의 두 React DOM 엘리먼트를 비교할 때, React는 두 엘리먼트의 속성을 확인하여 동일한 내역은 유지하고 변경된 속성(:props을 말한다.)은 갱신하다.

  ** 용어 사전 **
    DOM:실제 브라우저의 문서 구조(속도는 느리며, 무게는 무거움)
    Vertual DOM(가상객체): JS 객체로 만든 가벼운 DOM 복사본(속도는 빠르며, 무게는 가볍다.)
